Building a Custom AI

You can write a JavaScript AI and upload it to compete in Versus and Bot vs Bot modes. Your file runs in a sandboxed Web Worker — write it as a plain .js file with a single exported function, and the game handles the rest.

01 Quick Start

The fastest way to get started is to copy example-ai.js from the repo root. It already implements a working greedy one-piece lookahead AI.

  1. Copy or download example-ai.js.
  2. From the main menu, click VS AI or WATCH.
  3. Click MANAGE AI (or the AI Manager button), then + Upload .js AI file.
  4. Select your file — it appears in the list immediately.
  5. Select it and click Done, then start the game.

Your AI is stored in localStorage and persists across sessions. You can upload multiple AIs and switch between them in the AI Manager.

02 Required Interface

Your file must define a function named exactly getBestMove at the top level: 

function getBestMove(bot, pendingGarbage) {
  // Analyze bot state and return a move.
  return { rotationIndex, x, y, useHold };
}

Do not define self.onmessage — the game injects its own message handler that calls your function. Defining it yourself will break the integration.

Do not use ES module syntax (import / export). Write everything in a single self-contained plain JS file.

03 Input: the bot object

Called once per piece spawn. bot contains the full game state your AI needs:

Field Type Description
bot.board number[][] 20 rows × 10 columns. 0 = empty cell, non-zero = filled.
bot.active object The currently falling piece — see sub-fields below.
  .type string Piece letter: 'I' 'O' 'T' 'S' 'Z' 'J' 'L'
  .rotationIndex 0–3 Current rotation state of the spawned piece.
  .x number Column of the bounding-box left edge (spawn position).
  .y number Row of the bounding-box top edge (spawn position).
bot.nextQueue string[] Next 5 upcoming piece types (index 0 = next).
bot.hold string | null Piece currently in the hold slot, or null if empty.
bot.holdUsed boolean true if hold has already been used this turn (can't hold again).
bot.bagState string[] Remaining pieces in the current 7-bag. Useful for deep lookahead.
bot.lines number Total lines cleared so far this game.
bot.dead boolean true if the bot has topped out (game over).

board coordinate system

bot.board[0] is the top row, bot.board[19] is the bottom row. Columns run left (0) to right (9). A piece at x=3, y=0 is near the top-center of the board.

04 Input: pendingGarbage

A number representing garbage lines that will arrive at the bottom of your board soon. A value of 0 means no incoming garbage. Use this to decide whether to play aggressively (clear lines fast to cancel garbage) or stack normally.

05 Return value

Return a plain object with these four fields:

FieldTypeDescription
rotationIndex 0–3 Rotation state the piece should be in when it lands.
x 0–9 Column of the bounding-box left edge at the landing position.
y 0–19 Row of the bounding-box top edge at the landing position (after hard drop).
useHold boolean If true, the hold mechanic fires before the piece is placed.

The easiest way to find a valid y is to compute a hard-drop: drop the piece down from row 0 until it collides — that landing row is your y. See hardDropY() in the example.

06 Constraints

ConstraintDetail
2-second timeout If getBestMove does not return within 2 s, your worker is terminated and the game falls back to the built-in AI for the remainder of the match.
No network fetch, XMLHttpRequest, WebSocket, and importScripts are all blocked.
Single file All code must live in one .js file. No ES module imports.
No self.onmessage The game injects the worker message handler. Defining your own will break the integration.
Synchronous only getBestMove must return synchronously — async functions / Promises are not supported.

07 Helper functions you'll need

Because your file runs in isolation you must include your own board-manipulation utilities. These three are the essential building blocks — copy them from example-ai.js:

Piece rotation matrices

Each piece has 4 rotation states. Copy the PIECES constant from the example — it's a map of piece type → array of four 2D rotation grids.

collides(board, rotation, x, y)

Returns true if placing rotation at (x, y) overlaps a filled cell or goes out of bounds. Used by the other helpers. 

function collides(board, rotation, x, y) {
  for (let r = 0; r < rotation.length; r++) {
    for (let c = 0; c < rotation[r].length; c++) {
      if (!rotation[r][c]) continue;
      const row = y + r, col = x + c;
      if (col < 0 || col >= 10 || row >= 20) return true;
      if (row >= 0 && board[row][col] !== 0) return true;
    }
  }
  return false;
}

hardDropY(board, rotation, x)

Returns the lowest row a piece can reach by dropping straight down from row 0. 

function hardDropY(board, rotation, x) {
  let y = 0;
  while (!collides(board, rotation, x, y + 1)) y++;
  return y;
}

lockAndClear(board, rotation, x, y)

Returns a new board with the piece placed at (x, y) and all full rows removed. Also returns linesCleared for scoring. 

function lockAndClear(board, rotation, x, y) {
  const b = board.map(row => [...row]);
  for (let r = 0; r < rotation.length; r++)
    for (let c = 0; c < rotation[r].length; c++)
      if (rotation[r][c] && y + r >= 0) b[y + r][x + c] = 1;
  const kept = b.filter(row => row.some(v => v === 0));
  const cleared = 20 - kept.length;
  const empty = Array.from({ length: cleared }, () => new Array(10).fill(0));
  return { board: [...empty, ...kept], linesCleared: cleared };
}

08 Writing an evaluation function

Your evaluation function scores a board state — higher = better. A common starting set of heuristics:

HeuristicTypical weightNotes
Aggregate height −0.5 per unit Sum of all column heights. Penalises tall stacks.
Holes −0.8 per hole Empty cells with a filled cell directly above. Very bad — they're hard to clear.
Bumpiness −0.3 per unit Sum of absolute height differences between adjacent columns. Rewards flat boards.
Lines cleared +0 / +1 / +2.5 / +5 Reward clearing 2, 3, or 4 lines. Clearing 1 line alone is roughly break-even.

Tune the weights to change your AI's play style. Heavier hole penalties make the AI prioritise clean stacking; heavier line-clear bonuses encourage aggressive Tetris setups.

Using lookahead

For stronger play, simulate placements for the next piece too (two-piece lookahead). bot.nextQueue[0] is the next piece, bot.nextQueue[1..4] are the pieces after that, and bot.bagState contains the rest of the current 7-bag. Multi-piece search grows exponentially — stay well under 2 s.

09 Annotated example: example-ai.js

The full source with inline annotations:

// ── 1. Piece rotation matrices ──────────────────────────────────
// Required: the game doesn't provide these to you, so include them.
const PIECES = {
  I: [ /* 4 rotations × 4×4 grid */ ],
  T: [ /* 4 rotations × 3×3 grid */ ],
  // ... O, S, Z, J, L
};

// ── 2. Board utilities ──────────────────────────────────────────
// collides, hardDropY, lockAndClear — see Section 07 above.

// ── 3. Evaluation function ──────────────────────────────────────
// Returns a score — higher is better.
function evaluate(board, linesCleared) {
  // Compute column heights and hole count.
  // Return weighted sum of heuristics.
  return -0.5 * aggHeight + lineClearScore - 0.8 * holes - 0.3 * bumpiness;
}

// ── 4. Move finder ──────────────────────────────────────────────
// Try every (rotation, column) combination; return the best one.
function bestPlacementFor(board, pieceType) {
  let best = null, bestScore = -Infinity;
  for (let rot = 0; rot < 4; rot++) {
    const rotation = PIECES[pieceType][rot];
    for (let x = 0; x <= 10 - rotation[0].length; x++) {
      if (collides(board, rotation, x, 0)) continue; // column blocked at spawn
      const y = hardDropY(board, rotation, x);
      const { board: result, linesCleared } = lockAndClear(board, rotation, x, y);
      const score = evaluate(result, linesCleared);
      if (score > bestScore) { bestScore = score; best = { rotationIndex: rot, x, y, score }; }
    }
  }
  return best;
}

// ── 5. Main entry point ─────────────────────────────────────────
// The game calls this function each time a new piece spawns.
function getBestMove(bot, pendingGarbage) {
  const active = bestPlacementFor(bot.board, bot.active.type);

  // Optionally consider hold: if hold gives a better score, use it.
  if (!bot.holdUsed) {
    const holdType = bot.hold ?? bot.nextQueue[0];
    const fromHold = bestPlacementFor(bot.board, holdType);
    if (fromHold && active && fromHold.score > active.score + 0.5)
      return { ...fromHold, useHold: true };
  }

  if (active) return { ...active, useHold: false };

  // Fallback: stay in place (should rarely happen).
  return { rotationIndex: 0, x: bot.active.x, y: bot.active.y, useHold: false };
}

10 Tips for stronger play

  • Start with the example, verify it uploads and plays, then tune weights in evaluate().
  • Add a two-piece lookahead: for each candidate placement of the active piece, run bestPlacementFor on the resulting board with bot.nextQueue[0].
  • React to pendingGarbage > 4 by temporarily weighting line clears more heavily.
  • Keep a well column (one intentionally low column) for Tetris setups — the built-in AI does this explicitly.
  • Use console.log() freely — output appears in the browser DevTools console and doesn't affect play.
  • Profile with performance.now() to ensure you stay comfortably under the 2 s limit even on slow machines.
