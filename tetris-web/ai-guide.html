Building a Custom AI

You can write a JavaScript AI and upload it to compete in Versus and Bot vs Bot modes. Your file runs in a sandboxed Web Worker — write it as a plain .js file with a single exported function, and the game handles the rest.

01 Quick Start

The fastest way to get started is to download the example AI below — it already implements a working greedy one-piece lookahead AI. Then upload it as-is to verify everything works, and modify it from there.

⇓ Download example-ai.js

  1. Download example-ai.js using the button above.
  2. From the main menu, click AI MANAGER.
  3. Click + Upload .js AI file and select your file — it appears in the list immediately.
  4. Select it and click Done.
  5. Start a VERSUS, WATCH, or BOT VS BOT game — your AI will be used.

Your AIs are saved to your account and persist across sessions and devices. You can upload multiple AIs and switch between them in the AI Manager.

02 Required Interface

Your file must define a function named exactly getBestMove at the top level: 

function getBestMove(bot, pendingGarbage) {
  // Analyze bot state and return a move.
  return { rotationIndex, x, y, useHold };
}

Do not define self.onmessage — the game injects its own message handler that calls your function. Defining it yourself will break the integration.

Do not use ES module syntax (import / export). Write everything in a single self-contained plain JS file.

03 Input: the bot object

Called once per piece spawn. bot contains the full game state your AI needs:

Field Type Description
bot.board number[][] 20 rows × 10 columns. 0 = empty cell, non-zero = filled.
bot.active object The currently falling piece — see sub-fields below.
  .type string Piece letter: 'I' 'O' 'T' 'S' 'Z' 'J' 'L'
  .rotationIndex 0–3 Current rotation state of the spawned piece.
  .x number Column of the bounding-box left edge (spawn position).
  .y number Row of the bounding-box top edge (spawn position).
bot.nextQueue string[] Next 5 upcoming piece types (index 0 = next).
bot.hold string | null Piece currently in the hold slot, or null if empty.
bot.holdUsed boolean true if hold has already been used this turn (can't hold again).
bot.bagState string[] Remaining pieces in the current 7-bag. Useful for deep lookahead.
bot.lines number Total lines cleared so far this game.
bot.dead boolean true if the bot has topped out (game over).

board coordinate system

bot.board[0] is the top row, bot.board[19] is the bottom row. Columns run left (0) to right (9). A piece at x=3, y=0 is near the top-center of the board.

04 Input: pendingGarbage

A number representing garbage lines queued to arrive at the bottom of your board on your next lock. Each garbage row is a full row of filled cells with one random gap. Incoming garbage is cancelled by your own outgoing attack before anything hits your board — so clearing lines under pressure is doubly valuable.

Use pendingGarbage to adjust strategy: weight line clears more heavily when it is large (survival mode), and play normally or offensively when it is 0.

Threshold tip: the built-in hard AI shifts into garbage-clearing mode when pendingGarbage > 0. A simple heuristic: if pendingGarbage ≥ 4 (a Tetris worth of garbage), strongly prefer any placement that clears lines over shape-optimal placements.

05 Garbage & attack system

Every line clear sends a certain number of garbage lines to your opponent. Understanding this table is essential for building a competitive AI — maximising attack per piece is just as important as keeping a clean board.

Combo bonus

Clearing at least one line on consecutive pieces earns a cumulative combo bonus added on top of the base attack. The combo counter resets the moment you lock a piece without clearing. This game uses the Jstris combo table:

Cancellation

Your attack is subtracted from your incoming pendingGarbage first. Only the surplus reaches your opponent, and only the uncancelled remainder arrives on your board. Example: you have 6 incoming garbage and clear a Tetris (+4) — 4 lines are cancelled, 2 lines still arrive on your board, and nothing is sent to your opponent.

Efficiency note: T-spin doubles (4 lines sent for 2 lines cleared) and back-to-back Tetrises (5 lines sent for 4 cleared) are the most lines-sent-per-piece-used at high level. A sustained combo — even with doubles — can exceed both if the board stays clean.

06 Return value

Return a plain object with these four fields:

FieldTypeDescription
rotationIndex 0–3 Rotation state the piece should be in when it lands.
x 0–9 Column of the bounding-box left edge at the landing position.
y 0–19 Row of the bounding-box top edge at the landing position (after hard drop).
useHold boolean If true, the hold mechanic fires before the piece is placed.

The easiest way to find a valid y is to compute a hard-drop: drop the piece from row 0 until it collides — that landing row is your y. See hardDropY() in the example.

Bounding box x can be negative. Because the I piece uses a 4×4 grid with the vertical bar offset to column 1 or 2, a valid placement in board column 0 requires x = −1 or x = −2. Always use collides() to validate; never clamp x to ≥ 0.

08 Rotation states & SRS wall kicks

The four rotation states

Every piece has four rotation states numbered 0–3, cycling clockwise:

IndexNameReached by
0SpawnStarting orientation when a piece enters the board
1R (CW)One clockwise rotation from spawn
22 (180°)Two clockwise rotations from spawn
3L (CCW)Three clockwise rotations — same as one CCW from spawn

Your AI returns a rotationIndex (0–3) as the target state — the game handles actually spinning the piece there. You don't need to simulate individual rotation steps unless you're doing advanced move validation.

Bounding boxes

Each rotation is stored in a fixed-size 2D grid. The grid size varies by piece:

PieceGrid sizeNotes
I4×4Asymmetric — CW state occupies col 2, CCW occupies col 1
O2×2All four states are identical
T, S, Z, J, L3×3Standard bounding box

The x and y in bot.active and in your return value refer to the top-left corner of the bounding box, not the visible cells. This means x can be negative (e.g. an I piece hanging off the left edge stays in a valid column inside the grid). Always use collides() to validate a position rather than checking bounds manually.

Full rotation matrices

The exact matrices used by the game — copy these into your AI as the PIECES constant. Rows run top to bottom, columns left to right. 1 = filled cell.

SRS wall kicks

When the game executes a rotation, it first tries placing the piece with no offset. If that position collides, it tries up to four alternative offsets — called wall kicks — in order. The first offset that doesn't collide is used; if all five fail (including the no-offset attempt), the rotation is rejected.

For your AI this usually doesn't matter — you return a target {rotationIndex, x, y} and the game places the piece directly there. However, if you're generating candidate moves by iterating columns and want to know whether a particular rotation at a given x is reachable, you need to understand that some positions are only reachable via a kick (spinning into a gap), not by sliding in from the side.

Wall kick tables

Each entry is [dx, dy]. dx positive = right; dy positive = down (board Y increases downward). The game uses two separate tables:

J, L, S, T, Z — 5 kick offsets per transition:

// [dx, dy] — dy positive = down
const WALL_KICKS_JLSTZ = [
  // 0 → R  [[ 0, 0], [-1, 0], [-1,-1], [ 0, 2], [-1, 2]],
  // R → 2  [[ 0, 0], [ 1, 0], [ 1, 1], [ 0,-2], [ 1,-2]],
  // 2 → L  [[ 0, 0], [ 1, 0], [ 1,-1], [ 0, 2], [ 1, 2]],
  // L → 0  [[ 0, 0], [-1, 0], [-1, 1], [ 0,-2], [-1,-2]],
];

I piece — different offsets due to its 4×4 bounding box:

const WALL_KICKS_I = [
  // 0 → R  [[ 0, 0], [-2, 0], [ 1, 0], [-2,-1], [ 1, 2]],
  // R → 2  [[ 0, 0], [-1, 0], [ 2, 0], [-1, 2], [ 2,-1]],
  // 2 → L  [[ 0, 0], [ 2, 0], [-1, 0], [ 2, 1], [-1,-2]],
  // L → 0  [[ 0, 0], [ 1, 0], [-2, 0], [ 1,-2], [-2, 1]],
];

CCW rotation steps backwards through the table: rotating from state 1 (R) CCW means using the 0 → R kicks but applied in reverse (subtract the offset instead of adding). The O piece has no kick table — it never rotates in practice since all states are identical.

09 Helper functions you'll need

Because your file runs in isolation you must include your own board-manipulation utilities. These three are the essential building blocks — copy them from example-ai.js:

Piece rotation matrices

Copy the PIECES constant from section 08 above (or from example-ai.js).

collides(board, rotation, x, y)

Returns true if placing rotation at (x, y) overlaps a filled cell or goes out of bounds. Used by the other helpers. 

function collides(board, rotation, x, y) {
  for (let r = 0; r < rotation.length; r++) {
    for (let c = 0; c < rotation[r].length; c++) {
      if (!rotation[r][c]) continue;
      const row = y + r, col = x + c;
      if (col < 0 || col >= 10 || row >= 20) return true;
      if (row >= 0 && board[row][col] !== 0) return true;
    }
  }
  return false;
}

hardDropY(board, rotation, x)

Returns the lowest row a piece can reach by dropping straight down from row 0. 

function hardDropY(board, rotation, x) {
  let y = 0;
  while (!collides(board, rotation, x, y + 1)) y++;
  return y;
}

lockAndClear(board, rotation, x, y)

Returns a new board with the piece placed at (x, y) and all full rows removed. Also returns linesCleared for scoring. 

function lockAndClear(board, rotation, x, y) {
  const b = board.map(row => [...row]);
  for (let r = 0; r < rotation.length; r++)
    for (let c = 0; c < rotation[r].length; c++)
      if (rotation[r][c] && y + r >= 0) b[y + r][x + c] = 1;
  const kept = b.filter(row => row.some(v => v === 0));
  const cleared = 20 - kept.length;
  const empty = Array.from({ length: cleared }, () => new Array(10).fill(0));
  return { board: [...empty, ...kept], linesCleared: cleared };
}

10 Writing an evaluation function

Your evaluation function scores a board state — higher = better. A common starting set of heuristics:

HeuristicTypical weightNotes
Aggregate height −0.5 per unit Sum of all column heights. Penalises tall stacks.
Holes −0.8 per hole Empty cells with a filled cell directly above. Very bad — they're hard to clear.
Bumpiness −0.3 per unit Sum of absolute height differences between adjacent columns. Rewards flat boards.
Lines cleared −0.5 / +1 / +2.5 / +5 Single clears (1 line) slightly hurt board quality. Doubles onward are positive.

These weights optimise for clean survival. To play offensively — sending garbage via Tetrises or T-spins — you'll need additional heuristics: a well column bonus, a T-spin slot detector, and rewarding nearly-full rows that are one piece away from clearing.

Tune the weights to change play style. Heavier hole penalties produce very flat boards; higher line-clear bonuses drive Tetris or T-spin setups even at the cost of some board risk.

Using lookahead

For stronger play, simulate placements for the next piece too (two-piece lookahead). bot.nextQueue[0] is the next piece, bot.nextQueue[1..4] are the pieces after that, and bot.bagState contains the rest of the current 7-bag. Multi-piece search grows exponentially — stay comfortably under 2 s.

Responding to garbage

When pendingGarbage is large, your evaluation should temporarily favour placements that clear lines over placements that build setups. One practical approach: add pendingGarbage * linesCleared * 0.4 to the score so clearing under pressure becomes increasingly attractive as the threat grows.

11 Annotated example walkthrough

How the example AI fits together:

// ── 1. Piece rotation matrices ──────────────────────────────────
// Required: the game doesn't provide these to you, so include them.
const PIECES = {
  I: [ /* 4 rotations × 4×4 grid */ ],
  T: [ /* 4 rotations × 3×3 grid */ ],
  // ... O, S, Z, J, L
};

// ── 2. Board utilities ──────────────────────────────────────────
// collides, hardDropY, lockAndClear — see Section 09 above.

// ── 3. Evaluation function ──────────────────────────────────────
// Returns a score — higher is better.
function evaluate(board, linesCleared) {
  // Compute column heights and hole count.
  // Return weighted sum of heuristics.
  return -0.5 * aggHeight + lineClearScore - 0.8 * holes - 0.3 * bumpiness;
}

// ── 4. Move finder ──────────────────────────────────────────────
// Try every (rotation, column) combination; return the best one.
// x starts at -2 to handle I-piece vertical rotations whose bounding
// box extends left of the actual filled cells (see Section 08).
function bestPlacementFor(board, pieceType) {
  let best = null, bestScore = -Infinity;
  for (let rot = 0; rot < 4; rot++) {
    const rotation = PIECES[pieceType][rot];
    for (let x = -2; x <= 10 - rotation[0].length; x++) {
      if (collides(board, rotation, x, 0)) continue; // column blocked at spawn
      const y = hardDropY(board, rotation, x);
      const { board: result, linesCleared } = lockAndClear(board, rotation, x, y);
      const score = evaluate(result, linesCleared);
      if (score > bestScore) { bestScore = score; best = { rotationIndex: rot, x, y, score }; }
    }
  }
  return best;
}

// ── 5. Main entry point ─────────────────────────────────────────
// The game calls this function each time a new piece spawns.
function getBestMove(bot, pendingGarbage) {
  const active = bestPlacementFor(bot.board, bot.active.type);

  // Optionally consider hold: if hold gives a better score, use it.
  if (!bot.holdUsed) {
    const holdType = bot.hold ?? bot.nextQueue[0];
    const fromHold = bestPlacementFor(bot.board, holdType);
    if (fromHold && active && fromHold.score > active.score + 0.5)
      return { ...fromHold, useHold: true };
  }

  if (active) return { ...active, useHold: false };

  // Fallback: stay in place (should rarely happen).
  return { rotationIndex: 0, x: bot.active.x, y: bot.active.y, useHold: false };
}

const BOARD_ROWS = 20;
const BOARD_COLS = 10;

function collides(board, rotation, x, y) {
  for (let r = 0; r < rotation.length; r++)
    for (let c = 0; c < rotation[r].length; c++) {
      if (!rotation[r][c]) continue;
      const row = y + r, col = x + c;
      if (col < 0 || col >= BOARD_COLS || row >= BOARD_ROWS) return true;
      if (row >= 0 && board[row][col] !== 0) return true;
    }
  return false;
}

function hardDropY(board, rotation, x) {
  let y = 0;
  while (!collides(board, rotation, x, y + 1)) y++;
  return y;
}

function lockAndClear(board, rotation, x, y) {
  const b = board.map(row => [...row]);
  for (let r = 0; r < rotation.length; r++)
    for (let c = 0; c < rotation[r].length; c++)
      if (rotation[r][c] && y + r >= 0) b[y + r][x + c] = 1;
  const kept = b.filter(row => row.some(v => v === 0));
  const cleared = BOARD_ROWS - kept.length;
  const empty = Array.from({ length: cleared }, () => new Array(BOARD_COLS).fill(0));
  return { board: [...empty, ...kept], linesCleared: cleared };
}

function evaluate(board, linesCleared) {
  const heights = new Array(BOARD_COLS).fill(0);
  let holes = 0;
  for (let c = 0; c < BOARD_COLS; c++) {
    let found = false;
    for (let r = 0; r < BOARD_ROWS; r++) {
      if (board[r][c]) { if (!found) { heights[c] = BOARD_ROWS - r; found = true; } }
      else if (found) holes++;
    }
  }
  const aggHeight = heights.reduce((a, b) => a + b, 0);
  let bumpiness = 0;
  for (let c = 0; c < BOARD_COLS - 1; c++) bumpiness += Math.abs(heights[c] - heights[c+1]);
  const lineClearScore = [0, -0.5, 1.0, 2.5, 5.0][Math.min(linesCleared, 4)];
  return -0.5 * aggHeight + lineClearScore - 0.8 * holes - 0.3 * bumpiness;
}

function bestPlacementFor(board, pieceType) {
  let best = null, bestScore = -Infinity;
  for (let rot = 0; rot < 4; rot++) {
    const rotation = PIECES[pieceType][rot % 4];
    for (let x = -2; x <= BOARD_COLS - rotation[0].length; x++) {
      if (collides(board, rotation, x, 0)) continue;
      const y = hardDropY(board, rotation, x);
      const { board: result, linesCleared } = lockAndClear(board, rotation, x, y);
      const score = evaluate(result, linesCleared);
      if (score > bestScore) { bestScore = score; best = { rotationIndex: rot, x, y, score }; }
    }
  }
  return best;
}

function getBestMove(bot, pendingGarbage) {
  const active = bestPlacementFor(bot.board, bot.active.type);

  if (!bot.holdUsed) {
    const holdType = bot.hold ?? bot.nextQueue[0];
    const fromHold = bestPlacementFor(bot.board, holdType);
    if (fromHold && active && fromHold.score > active.score + 0.5)
      return { rotationIndex: fromHold.rotationIndex, x: fromHold.x, y: fromHold.y, useHold: true };
  }

  if (active) return { rotationIndex: active.rotationIndex, x: active.x, y: active.y, useHold: false };
  return { rotationIndex: 0, x: bot.active.x, y: bot.active.y, useHold: false };
}

13 Tips for stronger play

  • Start with the example, verify it uploads and plays, then tune weights in evaluate() until the play style feels right.
  • Add a two-piece lookahead: for each candidate placement of the active piece, run bestPlacementFor on the resulting board with bot.nextQueue[0]. This alone dramatically improves quality.
  • Keep a well column — one intentionally low column reserved for the I piece. Reward boards where exactly one column is meaningfully lower than its neighbours. This is how the built-in AI maintains a Tetris lane.
  • React to pendingGarbage > 4 by weighting line clears more heavily in evaluate(). Survival first, offensive play second.
  • For T-spin play: reward placements that leave a T-shaped gap with three corners filled (TSD/TST slot). The attack efficiency of a T-spin double matches a Tetris at 4 lines each, but requires only 2 rows of setup instead of 4.
  • Track combos yourself if you want to factor them in: after each line clear, increment a counter; reset it on any zero-clear lock. A 5-combo adds +2 to every clear's attack value — worth preserving if the board allows it.
  • Use console.log() freely — output appears in browser DevTools and doesn't affect play.
  • Profile with performance.now() to ensure you stay well under 2 s even on slow machines. A two-piece lookahead over all 7 pieces × 4 rotations × 10 columns (280 states per level) runs in microseconds — the budget is generous.
